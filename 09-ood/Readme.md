### Object-Oriented Design (OOD)?

Object-Oriented Design is the process of designing software by modeling real-world entities as objects and defining how they interact.

#### The Ultimate Goal of OOD

OOD isn't just about mapping real-world things to code. Its primary goals are:

- `Manage Complexity:` Break a massive system into smaller, self-contained pieces.
- `Increase Reusability:` Write code once and use it in multiple places.
- `Enable Scalability:` Make it easy to add new features without breaking old ones.
- `Improve Maintainability:` Isolate bugs and make changes without affecting the entire system.

#### The Four Pillars

1. Encapsulation
2. Abstraction
3. Inheritance
4. Polymorphism

#### The SOLID Principles

| Principle | Short Name                            | Goal                                                                   |
| --------- | ------------------------------------- | ---------------------------------------------------------------------- |
| **S**     | Single Responsibility Principle (SRP) | One class should have one responsibility.                              |
| **O**     | Open/Closed Principle (OCP)           | Open for extension, closed for modification.                           |
| **L**     | Liskov Substitution Principle (LSP)   | Child classes should replace parent classes without breaking behavior. |
| **I**     | Interface Segregation Principle (ISP) | Prefer many small interfaces over one large interface.                 |
| **D**     | Dependency Inversion Principle (DIP)  | Depend on abstractions, not concrete implementations.                  |

#### Unified Modeling Language - UML

It is a standardized, general-purpose visual modeling language used in software engineering to specify, visualize, construct, and document the artifacts of a software system.

#### Why is UML Used

1. **Communication:** It provides a common vocabulary for developers, product managers, designers, and stakeholders to discuss how the system should work.
2. **Visualization:** It helps teams visualize the structure and behavior of a system before writing a single line of code.
3. **Documentation:** It creates a permanent record of the system architecture that can be used for future maintenance or onboarding new team members.
4. **Complexity Management:** It breaks down large, complex systems into manageable, understandable pieces.
