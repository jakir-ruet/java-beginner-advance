### Design Pattern

Design patterns are reusable solutions to common software design problems. They represent best practices evolved over time by experienced developers and provide best practices for creating flexible, maintainable, and scalable applications.

#### Categories of Design Patterns

1. Creational Patterns - Object creation mechanisms
2. Structural Patterns - Class/object composition
3. Behavioral Patterns - Communication between objects

**1. Creational Patterns - These patterns deal with object creation mechanisms**

| Pattern          | Purpose                                        |
| ---------------- | ---------------------------------------------- |
| Singleton        | Ensures only one instance of a class exists    |
| Factory Method   | Creates objects without specifying exact class |
| Abstract Factory | Creates families of related objects            |
| Builder          | Constructs complex objects step-by-step        |
| Prototype        | Creates objects by cloning existing objects    |

**2. Structural Patterns - These patterns deal with class and object composition**

| Pattern   | Purpose                                       |
| --------- | --------------------------------------------- |
| Adapter   | Converts one interface into another           |
| Bridge    | Separates abstraction from implementation     |
| Composite | Treats individual and group objects uniformly |
| Decorator | Adds functionality dynamically                |
| Facade    | Provides a simplified interface               |
| Flyweight | Reduces memory usage                          |
| Proxy     | Controls access to an object                  |

**3. Behavioral Patterns - These patterns deal with communication between objects**

| Pattern                 | Purpose                                   |
| ----------------------- | ----------------------------------------- |
| Observer                | One-to-many notification mechanism        |
| Strategy                | Encapsulates algorithms                   |
| Command                 | Encapsulates requests as objects          |
| Iterator                | Sequentially accesses collection elements |
| Mediator                | Reduces direct dependencies               |
| State                   | Changes behavior based on state           |
| Template Method         | Defines algorithm skeleton                |
| Chain of Responsibility | Passes requests along a chain             |
| Visitor                 | Adds operations without modifying classes |
